# Slide 26 of Lecture 8 contains information about the L2-Regularization.

How to choose 位? In theory, as 'n' grows, 位 should be in the range O(1) to (n1/2). In practice, we optimize validation set or cross-validation error. This almost always decreases test error. Why use L2-Regularization? It "almost always decreases test error." The solution 'w' is unique, and there are no collinearity issues. The solution 'w' is less sensitive to changes in X or y. Gradient Descent converges faster, where bigger 位 means fewer iterations. In the worst case, we can just set 位 small and get the same performance.
